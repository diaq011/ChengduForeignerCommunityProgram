# 开发日志（Development Log）

## 文档用途

本文件记录**已落地到仓库**的开发进展，供团队成员与后续 AI 对话快速恢复上下文。  
与产品/架构设计文档不同，这里只写「做了什么、改在哪、怎么验、还剩什么」，不写需求原文。

**何时更新**

- 完成一个可描述的交付（功能、接口、页面、修复）后追加一条记录。
- 一次对话里有多项独立交付时，可拆成多条，或合并为一条并分小节。
- 仅讨论、未改代码的计划不必写入。

**新对话如何让 AI 读取**

1. 将本文件路径发给 AI：`docs/开发日志.md`
2. 说明当前负责的模块（如 Discover / Places / Events）。
3. 必要时再附上 `docs/仓库结构与三人协作说明.md` 与相关 OpenSpec change（`openspec/changes/<id>/`）。

**条目格式（复制模板使用）**

```markdown
## YYYY-MM-DD — 简短标题

| 字段 | 内容 |
|------|------|
| 模块 | mobile / admin / api / shared / docs / 跨端 |
| 负责人 | 可选：姓名或角色 |
| 关联 | 可选：OpenSpec change-id、Issue、PR |

### 摘要
一两句话说明本次交付。

### 主要变更
- 变更点 1
- 变更点 2

### 涉及路径
- `path/to/file`

### 验证
- 命令或手动步骤，以及结果（通过 / 未测）

### 未完成 / 下一步
- 可选
```

## 2026-06-03 — Discover：评论输入法回车误发送修复

| 字段 | 内容 |
|------|------|
| 模块 | mobile |
| 负责人 | Discover 模块 |
| 关联 | — |

### 摘要

修复帖子详情页评论框在中文输入法下：输入英文后按回车应确认组字上屏，却被当成「发送评论」的问题。

### 主要变更

- 评论框 `confirm-type` 由 `send` 改为 `done`。
- 增加 `compositionstart` / `compositionend` 处理；组字中及组字结束后短窗口内忽略键盘 `@confirm`。
- `@confirm` 延迟到下一 tick 再判断是否发送，避免与 IME 的 `compositionend` 竞态。

### 涉及路径

- `apps/mobile/src/pages/discover/detail.vue`

### 验证

- H5：`pnpm --filter @community-map/mobile dev:h5`，打开帖子详情，中文输入法输入英文后回车，应只上屏不发送；点击「发送」仍可发帖。

### 未完成 / 下一步

- 若需「非组字状态下回车发送」，需单独约定后再加（当前以按钮发送为主）。

## 2026-06-03 — Discover：小红书风格 UI 与交互优化

| 字段 | 内容 |
|------|------|
| 模块 | mobile |
| 负责人 | Discover 模块 |
| 关联 | — |

### 摘要

按产品反馈调整 Discover 三页视觉与交互：分类标签对齐与去黑边、举报弱化、评论输入固定底栏、整体参考小红书笔记流风格。

- **发帖页**：标签/语言选择与列表一致的芯片样式；主按钮「发布笔记」。

### 涉及路径

- `apps/mobile/src/pages/discover/index.vue`
- `apps/mobile/src/pages/discover/detail.vue`
- `apps/mobile/src/pages/discover/create.vue`

### 验证

- H5 本地预览三页布局与底栏固定行为。

### 未完成 / 下一步

- 双列瀑布流、点赞栏等更深的小红书交互未做。

## 2026-06-03 — Discover：契约、API、Mock、后台治理与移动端闭环

| 字段 | 内容 |
|------|------|
| 模块 | shared / api / mobile / admin |
| 负责人 | Discover 模块（契约需三人评审后合并） |
| 关联 | OpenSpec 个人开发计划（Discover 时间表） |

### 摘要

完成 Discover 内容流、发帖、详情评论、举报、标签筛选及后台帖子治理的 contract-first 闭环；默认仍使用 Mock 提供方。

### 主要变更

**packages/shared**

- 列表查询支持 `tagId`、`includeHidden`。
- 新增评论列表契约 `GET /discover/posts/:id/comments`（`CommentListQuerySchema`）。
- 评论创建支持 `parent_id`（为回复预留）。
- 客户端暴露 `discover.listComments`、`discover.reportPost`；`withQuery` 支持 boolean 查询参数。

**apps/api**

- Discover 路由与 Mock provider 实现上述契约；测试覆盖发帖 → 评论 → 举报 → 治理 → 公开列表不可见隐藏帖。

**apps/mobile**

- `discover/index.vue`：搜索、标签、分页、状态与卡片列表。
- `discover/create.vue`：表单、校验、标签、图片 URL 占位。
- `discover/detail.vue`：详情、评论列表、评论提交、举报。

**apps/admin**

- `PostsPage.vue`：`includeHidden` 拉取；保留 / 隐藏 / 删除操作。

### 涉及路径

- `packages/shared/src/schemas/discover.ts`、`contracts/discover.ts`、`client.ts`、`mock/service.ts` 等
- `apps/api/src/routes/discover.ts`、`providers/mock/index.ts`
- `apps/mobile/src/pages/discover/*.vue`
- `apps/admin/src/pages/PostsPage.vue`

### 验证

- `pnpm test`（Vitest，含 shared client/contracts 与 api app 测试）
- 各包 `typecheck` 通过
- 本地：`pnpm --filter @community-map/mobile dev:h5`（默认 Mock）；联调 API 时 `API_PROVIDER=mock pnpm --filter @community-map/api dev` + `VITE_API_MODE=http`

### 未完成 / 下一步

- CloudBase 真实存储与 `discover` 云函数/BFF 落地（provider 仍为 stub/部分 mock）。
- 图片真实上传（CloudBase `public/posts/`），当前为 URL 占位。
- 点赞、消息通知等与 Discover 相关的扩展接口。
- 微信真机/开发者工具对地图、评论 IME 的回归（H5 已测部分场景）。

## 维护约定（给 AI）

- 用户或监督流程要求「记录进度」时：**只追加**新条目到「日志条目（新在上）」区，不要删除旧条目；重大更正用新条目说明并引用日期。
- 条目必须可核对：至少列出关键文件路径和验证命令。
- 不要在本文件写密钥、token、内网地址；环境说明放在未来的 `CloudBase 环境配置说明.md`。
- 与 OpenSpec 任务勾选、`progress.txt` 的关系：OpenSpec 管**单次 change 的执行审计**；本文件管**对人友好的跨会话项目进展摘要**。
